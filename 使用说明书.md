## 一、工具简介

### 这个工具做什么？

上传每月4-5个Excel文件（发票、银行流水、工资表），自动生成《小企业会计准则》规定的三张报表：

| 报表 | 代码 | 用途 |
|------|------|------|
| 利润表 | PL | 看赚了多少 |
| 资产负债表 | BS | 看有多少家底、欠多少债 |
| 现金流量表 | CF | 看钱从哪里来、花到哪里去 |

另外还能自动生成：
- **Word 分析报告** — 含7大板块的财务分析，自动标出风险
- **多期对比表** — 选任意月份横向对比，自动标异常
- **自定义模板填充** — 上传你自己的Excel模板，自动填数

### 安全吗？

- ✅ 所有计算在你电脑上完成
- ✅ 没有任何后端服务器，不上传数据
- ✅ 不需要联网
- ✅ 数据存在 `data/` 目录，随时可备份

### 哪些人适用？

小企业会计、代账会计。只要你会用浏览器、会导Excel文件。

## 二、安装与启动

### 2.1 首次安装（需要联网，只装一次）

以管理员身份打开命令提示符（cmd），输入：

```bash
pip install streamlit openpyxl xlrd python-docx pandas
```

等待几分钟，显示 `Successfully installed` 即完成。

### 2.2 启动工具

双击项目文件夹里的 **`启动报表工具.bat`**，会自动打开浏览器并显示工具界面。

如果没有 bat 文件，也可以在项目目录打开命令提示符，输入：

```bash
streamlit run app.py
```

打开浏览器访问 `http://localhost:8501`。

### 2.3 关闭工具

直接关闭命令提示符窗口即可。所有数据自动保存，不会丢失。

## 三、首次配置

> ⚠️ 首次配置只需做一次。以后每月直接用。

### 步骤1：创建公司

1. 左侧导航点 **「公司档案」**
2. 输入公司全称（如：来宾市顺恒废旧汽车回收有限公司）
3. 点「创建新公司」

### 步骤2：填写年初余额

在「公司档案」页面：

1. 切换到 **「年初余额」** 标签
2. 找到公司去年12月的资产负债表（期末余额列）
3. 把以下项目的期末数照抄进去：

**必须填写：**
- 货币资金（银行存款余额合计）
- 应收账款
- 预付账款
- 其他应收款
- 存货
- 固定资产原价、累计折旧
- 长期待摊费用
- 应付账款
- 预收账款
- 应付职工薪酬
- 应交税费
- 其他应付款
- 实收资本
- 未分配利润

> 💡 **提示**：如果不填年初数，资产负债表年初余额列全为0，但不影响利润表生成。填了才能自动算平资产负债表。

### 步骤3：配置科目映射

1. 左侧导航点 **「科目映射」**
2. 选择报表类型（利润表/资产负债表/现金流量表）
3. 系统已预置常用科目映射，你可以直接使用，也可以修改

**如果需要添加科目：**
1. 选「添加映射规则」
2. 输入科目编码（金蝶里的编码，如 500101）
3. 选择对应的报表行（如"营业成本"）
4. 选规则类型：
   - `code_match`：源数据里的科目编码→报表行（最常用）
   - `auto_calc`：由计算引擎自动产出（折旧、税费等）
   - `multi_source`：多个来源汇总（如管理费用 = 发票+工资+社保+折旧）

> 💡 **提示**：上传一份历史发票可以快速发现有哪些科目编码。工具会列出所有编码，你只需要给每个编码选对应报表行。

### 步骤4：完成！

首次配置完成。以后每月只需要3步。

## 四、每月操作

### 准备文件（每月必做）

每个月你需要准备5个文件：

| 文件名 | 内容 | 格式要求 |
|--------|------|----------|
| `sales.xlsx` | 本月销售收入发票 | 金蝶导出，sheet名"信息汇总表" |
| `costs.xlsx` | 本月成本费用发票 | 同上 |
| `nong.xls` | 农行银行流水 | 原格式导出 |
| `xinyong.xls` | 信用社银行流水 | 原格式导出 |
| `payroll.xlsx` | 本月工资表 | 含"合计"行 |

### 操作步骤（约3分钟）

**步骤1 — 导入数据**

点 **「数据导入」** → 依次上传5个文件 → 确认

**步骤2 — 核对数据**

自动跳到 **「数据核验」** 页面：
- 看一眼收入、成本、工资数字对不对
- 系统自动检查科目编码是否有新增
- 确认无误，点「确认无误，生成报表」

**步骤3 — 生成报表**

点 **「报表生成」** → 点「🚀 生成三大报表」

自动产出：
- 利润表 xlsx
- 资产负债表 xlsx
- 现金流量表 xlsx
- 自动保存历史摘要（用于后续对比）

**步骤4 — 下载**

点 **「分析导出」**：
- 下载三张报表 xlsx
- 点「生成分析报告」→ 下载 Word 分析报告

每月操作完成！

## 五、各页面详解

### 5.1 首页

显示快速操作入口，以及当前选中公司的基本信息。

### 5.2 公司档案

- **基本信息**：公司全称（用在报表表头）
- **年初余额**：去年12月31日的资产负债表数值，填一次管一年

### 5.3 科目映射

核心配置页面。决定了发票上的科目编码对应到报表的哪一行。

**报表行列表**：系统预置了标准小企业会计准则的报表行。你在左侧选一个科目编码，右侧下拉选它属于哪个报表行。

**三种规则类型**：
- `code_match`：普通科目编码→报表行（如 5001→营业成本）
- `auto_calc`：由计算引擎根据公式自动产出（如折旧=月折旧额×月数）
- `multi_source`：某个报表行由多个来源拼成（如管理费用）

### 5.4 数据导入

上传5个文件。文件命名必须严格：

| 文件名 | 说明 |
|--------|------|
| `sales.xlsx` | 销项发票 |
| `costs.xlsx` | 进项/费用发票 |
| `nong.xls` | 农行对账单 |
| `xinyong.xls` | 信用社对账单 |
| `payroll.xlsx` | 工资表 |

上传后系统自动解析每个文件的内容并显示摘要。

### 5.5 数据核验

系统自动检查：
- **文件完整性**：5个文件是否齐全
- **科目编码**：是否有新出现的科目编码（需要去映射页添加）
- **数据合理性**：发票数、银行余额等是否在合理范围

核验通过后点「确认无误，生成报表」。

### 5.6 报表生成

**单月生成**：生成当前选中月份的三大报表。

**自定义模板（可选）**：
- 展开「📋 自定义模板」
- 上传你自己的 Excel 报表模板
- 系统自动识别模板里每一行对应什么数据
- 生成时优先使用你的模板填充

**批量生成**：
- 如果有多个月的数据已上传，系统自动检测
- 勾选要生成的月份，一键批量产出

### 5.7 分析导出

**下载报表**：三张报表的 Excel 文件

**生成分析报告**：
- 点「生成分析报告」→ 等待几秒 → 下载 Word 文档
- 报告包含7大板块：编制基础、经营情况、资产负债、现金流量、税务、综合评估、建议
- 自动检测风险（资不抵债、关联方依赖、现金流薄弱等）并用红色标注

### 5.8 历史对比

- 自动扫描历史生成过的月份
- 多选2个以上月份即可横向对比
- 两个月份时自动显示变动额和变动率
- 变动超过30% ⚠ 警告，超过50% 🔴 严重
- 3个月以上自动显示趋势方向（上升/下降/持平）
- 支持导出对比表为 Excel

## 六、高级功能

### 6.1 批量生成

适用于：一次处理多个月份（如年后补1-5月）。

1. 在「数据导入」页，按月分别上传各月数据（自动存到不同目录）
2. 在「报表生成」页，批量生成区域会列出所有可用月份
3. 勾选需要的月份，点「批量生成」
4. 进度条实时显示，几分钟全搞定

### 6.2 自定义模板

适用于：公司有自己的报表格式，不想用系统预置格式。

1. 在「报表生成」页展开「📋 自定义模板」
2. 上传你的 Excel 报表模板（利润表/资产负债表/现金流量表各一个）
3. 系统自动解析模板结构，模糊匹配行标签
4. 查看匹配结果，确认无误
5. 之后每次生成报表都会用你的模板填充数据

> 💡 **原模板的格式（字体、边框、颜色、合并单元格）全部保留，系统只填数字。**

### 6.3 历史对比

适用于：分析月度经营趋势、向老板汇报。

1. 选「历史对比」页面
2. 勾选要对比的月份（至少2个）
3. 系统显示利润表和资产负债表的所有关键指标横向排列
4. 自动标注异常波动
5. 可导出 Excel 对比表

## 七、数据备份

### 备份什么？

整个 `data/` 目录。它包含：

```
data/
├── companies/        公司配置和科目映射
├── uploads/          上传的源文件（发票、银行、工资）
├── output/           生成的报表和分析报告
└── templates/        自定义模板
```

### 如何备份？

直接复制整个 `data/` 文件夹到U盘或网盘即可。

### 如何恢复？

把备份的 `data/` 文件夹覆盖回项目目录即可。

> 💡 **建议每月生成完报表后做一次备份。**

## 八、常见问题

### Q1：为什么"资产负债表不平"？

检查三件事：
1. 年初余额是否完整填写（特别是未分配利润）
2. 科目映射是否把所有发票科目都配了报表行
3. 是否有新增科目编码未映射（看核验页面的提示）

**原因**：资产负债表里的"其他应付款"是倒挤出来的（plug figure），会强制让表平。如果它还平不了，说明年初余额或映射有遗漏。

### Q2：利润表里"营业收入"是0？

检查：
- `sales.xlsx` 是否上传正确
- 发票 sheet 名是否为"信息汇总表"（金蝶默认）
- 科目映射里营业收入对应的科目编码是否配置了

### Q3：银行余额和银行对账单对不上？

可能原因：
- 银行文件不是标准格式（农行从第3行开始，信用社从第4行开始）
- 如果有其他银行格式，需要修改 `data_extractor.py` 适配

### Q4：系统提示"发现新的科目编码"

去「科目映射」页面，给新出现的科目编码选一个报表行。这是正常情况，说明这个月有新的费用类型。

### Q5：如何用自己公司的报表格式？

在「报表生成」页展开「📋 自定义模板」，上传你的 Excel 模板即可。系统会自动匹配行标签并填入数据。

### Q6：数据存在哪里？

所有数据在 `data/` 目录。这是唯一需要备份的目录。其他 `.py` 文件都是程序代码，不需要备份。

### Q7：可以多个公司共用吗？

可以。左侧「当前公司」下拉框切换公司。每个公司有独立的配置、映射、数据、报表。

### Q8：一台电脑上能用，另一台能用吗？

可以。复制整个项目文件夹到另一台电脑，运行 `pip install -r requirements.txt` 安装依赖，然后双击启动即可。如果两台电脑都要用，建议把 `data/` 目录放在U盘。

### Q9：会计科目编码变了怎么办？

去「科目映射」页修改对应科目的映射规则即可。旧的映射可以删除，新的添加进去。不影响历史报表。

### Q10：现金流量表的数据从哪里来？

现金流量表用**间接法**编制：
1. 从净利润出发
2. 加回非现金支出（折旧、摊销）
3. 调整营运资金变动（应收、应付、存货的变化）
4. 加上投资和筹资活动现金流

数据来源是银行流水的实际收支分类 + 利润表和资产负债表的数据变化。

## 附录：文件格式说明

### 发票文件 (sales.xlsx / costs.xlsx)

金蝶标准导出格式：
- Sheet名：`信息汇总表`
- 第1行：表头
- 数据从第2行开始
- 列8（I列）：日期
- 列9（J列）：科目编码
- 列11（L列）：货品名称
- 列16（Q列）：金额（不含税）
- 列18（S列）：税额

### 银行流水 (nong.xls / xinyong.xls)

**农行格式：**
- 数据从第3行开始
- 列0：日期 | 列1：收入 | 列2：支出 | 列3：余额 | 列7：摘要

**信用社格式：**
- 数据从第4行开始
- 列0：日期 | 列1：支出 | 列2：收入 | 列3：余额

### 工资表 (payroll.xlsx)

- 必须有一行第1列是"合计"
- 第5列（F列）：应发工资合计
