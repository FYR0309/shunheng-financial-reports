# 财务报表生成工具 — 本地Web应用设计文档

> **设计日期：** 2026-06-29
> **状态：** 待审阅
> **目标：** 将顺恒废旧公司报表生成脚本改造为通用本地Web应用，供多家公司财务人员安全、准确地生成小企业会计准则财务报表

---

## 一、项目背景与目标

### 1.1 现状

目前 `generate_formatted.py` 已能为"来宾市顺恒废旧汽车回收有限公司"生成符合模板格式的三大报表（利润表、资产负债表、现金流量表）及Word分析报告。但脚本存在以下局限性：

- 报表格式硬编码（行项目名称、列宽、合并单元格写入代码）
- 科目编码（`1110701000000000000` 等）写死在代码中
- 换一家公司需要重写整份脚本
- 非技术人员无法操作（需要命令行 + 编辑 Python 代码）
- 没有数据核验环节，错误只能在最终报表中发现

### 1.2 目标

构建一个**本地运行的Web应用**，财务人员通过浏览器操作，实现：

1. 支持多公司档案管理，一套工具服务多家企业
2. 用户上传模板 + 配置科目映射，无需改代码
3. 每月常规操作3分钟内完成（导入数据 → 核验 → 生成 → 下载）
4. 所有数据处理在本地完成，数据不上传任何服务器
5. 内置多层验证，阻止错误数据进入最终报表

### 1.3 非目标（明确不做）

- 不做云端SaaS（数据安全是硬约束）
- 不做自动读取金蝶/用友数据库（来源系统太多，先支持Excel上传）
- 不做税务申报对接（各地方税务系统不同）
- 不做多用户权限管理（单机本地工具，使用者即为操作者）

## 二、技术选型

| 层次 | 选择 | 理由 |
|------|------|------|
| 前端框架 | **Streamlit** | 原生支持DataFrame展示、文件上传/下载、表格预览；比Gradio更适合数据密集型界面；`st.sidebar` 天然支持导航 |
| 后端语言 | **Python 3.10+** | 现有代码可直接复用；openpyxl/xlrd/python-docx生态成熟 |
| 模板读取 | **xlrd + openpyxl** | xlrd读.xls模板格式，openpyxl生成.xlsx输出 |
| 文档生成 | **python-docx** | 现有 `generate_analysis_docx.py` 已验证可行 |
| 数据存储 | **JSON + 本地文件系统** | 公司配置、科目映射存在JSON；原始数据和输出存在文件系统；无需数据库 |
| 部署方式 | `streamlit run app.py` | 单命令启动，无需Nginx/WSGI；适合财务人员在本地电脑运行 |

### 3.1 模块职责

| 模块 | 文件 | 职责 |
|------|------|------|
| 模板引擎 | `template_engine.py` | 解析用户上传的Excel模板：自动识别表头区/数据区/金额列；支持合并单元格；输出标准化模板配置JSON |
| 科目映射器 | `mapping_engine.py` | 管理源数据科目编码 ↔ 报表行项目的对应关系；支持科目编码匹配、自动计算、多源汇总三种规则 |
| 数据抽取器 | `data_extractor.py` | 从发票Excel、银行流水.xls、工资薪金表读取原始数据，输出标准化中间数据结构（与会计逻辑无关） |
| 计算引擎 | `calc_engine.py` | 执行会计计算：利润表汇总、资产负债表面计算、间接法现金流量表；使用科目映射配置驱动 |
| 格式渲染器 | `format_renderer.py` | 按模板配置精确写出Excel文件（列宽、合并单元格、字体、边框、数字格式）；从现有 `generate_formatted.py` 提取格式化逻辑 |
| 自检验证 | `validator.py` | 多层验证：数据格式→金额核对→平衡检查→三表勾稽；输出通过/失败/警告 |
| Streamlit UI | `app.py` | 所有页面逻辑；不包含业务计算，只做输入/展示/调用 |

### 3.2 关键设计决策

**存储目录使用 `./data/`（程序同级），不使用 `~/.financial-reports/`**

原因：可移植性好（整个文件夹拷贝即可迁移）；用户一眼知道数据在哪；多用户场景避免串数据。

**数据核验台是必经步骤**

数据导入后自动跳转核验台，未确认核验结果不能点击"生成报表"。这不是限制用户，是保护财务人员的职业安全。

## 四、模板引擎

### 4.1 预置 + 自定义双轨制

- **预置模板**：内置《小企业会计准则》标准格式（利润表、资产负债表、现金流量表），与目前顺恒使用的模板格式完全一致。全国小企业会计制度报表格式是统一的，90%以上的小企业用户开箱即用。
- **自定义模板**：用户上传自己公司的空报表或历史报表Excel，引擎自动解析行项目结构。适配有特殊格式需求的10%用户。

### 4.2 解析流程

```
步骤1: 上传模板文件（.xls 或 .xlsx）
        ↓
步骤2: 引擎自动扫描：
        - 识别标题区行号（连续空行之前的所有行）
        - 识别数据起始行（第一个包含数字"1"的行次所在行）
        - 识别"项目名称"列（包含中文最长的列）
        - 识别"行次"列（包含数字1,2,3...的列）
        - 识别"金额"列（标题含"本期/期末/累计/余额"的列）
        ↓
步骤3: 用户在界面上确认/修正自动识别结果：
        - 调整数据起始行
        - 指定哪一列是项目名称列
        - 勾选哪些列是金额列
        - 预览提取出的行项目列表
        ↓
步骤4: 保存为模板配置JSON
```

### 4.3 模板配置数据结构

```json
{
  "template_type": "利润表",
  "excel_file": "顺恒2025年12月_利润表.xls",
  "header_rows": [1, 2, 3, 4],
  "data_start_row": 5,
  "columns": {
    "item_name": 0,
    "row_num": 1,
    "value_cols": [2, 3]
  },
  "value_col_labels": ["本年累计数", "本月数"],
  "merge_ranges": [
    {"cells": "A1:D1", "type": "title"},
    {"cells": "A2:D2", "type": "subtitle"}
  ],
  "items": [
    {"row": 5, "name": "一、营业收入", "row_num": "1"},
    {"row": 6, "name": "减：营业成本", "row_num": "2"},
    {"row": 7, "name": "税金及附加", "row_num": "3"}
  ]
}
```

## 五、科目映射器

### 5.1 三种匹配规则

| 规则类型 | 说明 | 示例 |
|---------|------|------|
| **科目编码匹配** | 根据发票上的科目编码，归类到报表行 | `1110701000000000000` → 营业成本 |
| **自动计算** | 不依赖源数据，由计算引擎产出 | 折旧（月固定25,886.46）、税金及附加（增值税×12%） |
| **多源汇总** | 一个报表行汇总多个不同来源 | 管理费用 = 费用发票 + 工资薪金 + 社保 + 折旧 |

### 5.2 向导式配置流程（首次使用，一次性）

```
步骤1: 选择报表模板（预置或上传）
步骤2: 上传一份历史源数据（用于扫描所有出现的科目编码）
步骤3: 界面展示两栏：
       左栏 — 所有出现的科目编码（自动列出）
       右栏 — 报表行项目（从模板提取）
       用户用下拉菜单将左侧每个科目编码分配到右侧报表行
步骤4: 标注哪些报表行是"自动计算"类型（无需匹配源数据）
步骤5: 保存 mappings.json
```

### 5.3 特殊数据源处理

**工资表**（无科目编码）：通过列名关键词识别
```
"应发" / "应发合计" / "应发工资"    → 管理费用-工资
"社保" / "单位社保"                  → 管理费用-社保
"个税" / "个人所得税"                 → 应交税费
```

**银行流水**：不参与科目映射。用于：
- 现金流量表生成
- 货币资金余额验证
- 关联方借款变动倒推

### 5.4 核心安全原则

**不做任何科目编码归属的智能猜测。** 宁可让用户手动拖一遍，也不能自动把成本归到收入里。自动化的边界是"省重复劳动"，不碰"会计判断"。

## 六、数据抽取器

### 6.1 抽取器清单

| 抽取器 | 输入 | 输出结构 |
|--------|------|---------|
| `InvoiceExtractor` | 销售收入/成本费用.xlsx | `{月份: [{日期, 发票号, 科目编码, 金额}]}` |
| `BankExtractor` | 农行/信用社明细.xls | `[{日期, 收入, 支出, 余额, 银行名称}]` |
| `PayrollExtractor` | 工资薪金.xlsx | `{月份: {应发工资, 单位社保, 个人社保, 个税}}` |

### 6.2 关键处理逻辑

- **银行流水合并**：农行（正序）+ 信用社（倒序）→ 按日期统一排序
- **发票日期过滤**：只取报表期间内的发票；非当期排除并提示用户
- **工资表列名模糊匹配**：允许"应发工资"/"应发合计"/"实发工资"等多种列名变体
- **每个抽取器独立可测试**：输入文件路径，输出标准dict结构，不依赖全局状态

## 七、计算引擎

### 7.1 核心方法

```python
class ReportCalculator:
    def calculate_pl(extracted_data, mappings, period) -> PLResult
    def calculate_bs(extracted_data, mappings, opening_bs, pl_result, bank_balance) -> BSResult
    def calculate_cf(extracted_data, pl_result, bs_result, bank_data) -> CFResult
```

### 7.2 计算逻辑（沿用现有 generate_formatted.py 验证的逻辑）

**利润表：**
- 营业收入 = 汇总映射到"营业收入"的所有科目金额
- 营业成本 = 汇总映射到"营业成本"的所有科目金额
- 税金及附加 = 应纳增值税 × 12%（城建7%+教育3%+地方2%）
- 管理费用 = 费用发票 + 工资应发 + 单位社保 + 折旧 + 摊销 + 银行费
- 财务费用 = 银行账户管理费
- 净利润 = 营业收入 - 营业成本 - 税金及附加 - 管理费用 - 财务费用

**资产负债表：**
- 货币资金期末 = 银行实际余额（与现金流量表勾稽验证）
- 固定资产净值 = 期初 - 月折旧 × 月份数
- 长期待摊费用 = 期初 - 月摊销 × 月份数
- 应交税费 = 期初 + 本期计提 - 本期缴纳
- 未分配利润 = 期初 + 本期净利润
- **其他应付款 = 负债和所有者权益合计 - 所有已知项目之和**（倒挤，标注"待确认"）

**现金流量表（间接法）：**
- 经营CF = 净利润 + 折旧 + 摊销 - 经营性应付项目变动 - 经营性应收项目变动
- 投资CF = 购建固定资产支出
- 筹资CF = 关联方借款净增加（= -其他应付款减少额）
- 期末现金 = 期初现金 + 经营CF + 投资CF + 筹资CF = 银行实际余额

### 7.3 设计决策

**其他应付款倒挤后标注"待确认"**：用户可在界面上手动输入实际数覆盖。如果用户不修改，默认使用倒挤值。这样既保持灵活性，也提醒用户注意该数字的性质。

**现金流量表双轨输出**：默认生成标准间接法（符合准则）；额外利用银行流水产出简易收支分类汇总（非正式，帮用户直观理解钱花哪了）。

# 字体
font_title = Font(name='宋体', size=14)       # 报表标题

## 九、错误处理与验证体系

### 9.1 四层验证

```
数据导入 ──→ 核验台 ──→ 计算引擎 ──→ 报表输出
  │              │            │            │
格式检查      金额核对     平衡检查     三表勾稽
文件完整性    科目匹配     借贷平衡     交叉验证
```

### 9.2 各层检查详表

**第一层：数据导入时**

| 检查项 | 判断方式 | 级别 |
|--------|---------|------|
| 文件扩展名不正确 | .xls/.xlsx 检查 | 🔴 阻止 |
| 必填列缺失（如发票文件无"科目编码"列） | 扫描表头 | 🔴 阻止 |
| 发票日期不在当前期间 | 日期范围比对 | ⚠️ 警告（跳过该条） |
| 金额列包含空值或非数字 | 类型检查 | ⚠️ 警告（跳过该条） |

**第二层：核验台**

| 检查项 | 判断方式 | 级别 |
|--------|---------|------|
| 发票数量比上月偏差 >30% | 与历史数据比对 | ⚠️ 警告 |
| 存在未配置映射的新科目编码 | 查 mappings.json | 🔴 阻止 |
| 银行余额为负数 | 值检查 | 🔴 阻止 |
| 工资总额比上月偏差 >50% | 与历史数据比对 | ⚠️ 警告 |

**第三层：计算引擎**

| 检查项 | 级别 |
|--------|------|
| 资产负债表不平（A-L-E≠0） | 🔴 阻止 |
| 期末现金 ≠ 银行实际余额 | 🔴 阻止 |
| 净利润 ≠ 未分配利润变动额 | 🔴 阻止 |

**第四层：报表输出**

| 检查项 | 级别 |
|--------|------|
| 三表间勾稽关系不匹配 | 🔴 阻止 |
| 毛利率异常波动（与历史均值偏差>20个百分点） | ⚠️ 警告 |
| 管理费用率异常波动 | ⚠️ 警告 |

### 9.3 错误信息格式

每个错误必须包含三要素：

> **❌ 发现问题：** [具体数值/描述]
> **🔍 可能原因：** [给出1-2种常见原因]
> **🔧 建议操作：** [指向具体操作步骤]

### 9.4 审计日志

每次生成报表自动保存日志到 `data/logs/YYYY-MM-DD-公司名-操作.json`，包含：
- 源文件SHA256哈希
- 使用的模板和映射配置版本
- 中间计算结果
- 自检结果
- 时间戳

用于审计追溯——三个月后可以查"这个数当时是怎么来的"。

| **数据核验** | 预览抽取结果；金额确认；异常提示 | 每月（必经） |
| **报表生成** | 一键生成三表，在线预览，自检结果展示 | 每月 |
| **分析导出** | 下载Excel报表 + Word分析报告 | 每月 |

### 10.3 设计决策

- **首页两个核心按钮**够用，不做仪表盘。小企业会计不需要花哨图表。
- **核验台是必经步骤**，不确认不能生成报表。
- **侧边栏使用纯文字**，不用图标。加载快、维护简单。

## 十四、用户操作步骤（大白话版）

> 以下说明将随应用一同提供给最终用户。

### 第一次用（设置，只需做一次，约10分钟）

**第1步：打开工具**
> 双击桌面上的"财务报表工具"，浏览器会自动打开一个页面。

**第2步：创建你公司的档案**
> 点左边菜单的「公司档案」→「新建公司」→ 填上公司名字、税号 → 点保存。

**第3步：选报表格式**
> 系统内置了《小企业会计准则》的标准格式。如果你的公司按这个准则做账，直接点「使用预置模板」就行。
> 如果你的报表格式比较特殊，点「上传自己的模板」，把你以前的空报表Excel传上来。

**第4步：配科目编码（最关键的一步）**
> 点左边「科目映射」→ 上传一份**历史数据**（比如上个月的发票Excel）→ 系统会自动把你用到的所有科目编码列出来 → 你只需要把每个编码**下拉选**到对应的报表项目上。比如：
> - 哪些编码是"营业收入"？
> - 哪些编码是"营业成本"？
> - 哪些编码是"管理费用"？
>
> 做一遍就行，以后每个月自动记住。

**第5步：填年初数**
> 点「公司档案」→「年初余额」→ 把你去年12月资产负债表上的期末数填进去。系统需要这个数字来算本期变动。填一次就行。

> ✅ 设置完成！以后每个月只用3分钟。

---

### 每个月用（常规操作，约3分钟）

**第1步：导入数据（1分钟）**
> 打开工具 → 确认公司名字和月份是对的 → 点「数据导入」→ 把这4个文件拖进来：
> 1. 这个月的销售收入发票Excel
> 2. 这个月的成本费用发票Excel
> 3. 这个月的银行流水（农行和信用社各一份）
> 4. 这个月工资表
>
> 点「下一步」。

**第2步：核对数据（1分钟）**
> 系统会展示它读到了什么：
> - 销售收入多少张发票，一共多少钱
> - 成本费用多少张，一共多少钱
> - 银行最后余额是多少
> - 工资发了多少
>
> 你扫一眼，确认数字跟你手上的对得上。如果有红色警告，按提示改。没问题就点「确认无误」。

**第3步：生成报表（30秒）**
> 点「生成报表」→ 三张表（利润表、资产负债表、现金流量表）自动生成 → 系统自动检查有没有错误 → 在线预览，确认没问题。

**第4步：下载（30秒）**
> 点「下载Excel报表」保存三张表。
> 点「下载Word分析报告」保存分析说明。
>
> 搞定。

### 碰到问题怎么办

| 提示 | 意思 | 怎么做 |
|------|------|--------|
| 🔴 "发现新的科目编码未配置" | 这个月出现了一个新的科目编码，系统不知道归到哪一行 | 去「科目映射」页，给它归个类就行 |
| 🔴 "资产负债表不平" | 数字对不上，通常是有新科目没配置 | 回到科目映射检查，或者联系系统管理员 |
| ⚠️ "本月发票数比上月少很多" | 提醒你确认是不是漏传了文件 | 检查是不是所有发票都放进去了 |
| ⚠️ "毛利率异常波动" | 这个月赚的比例跟以前差很多 | 可能是正常的淡旺季，也可能是成本或收入记错了，看一眼 |

所有提示都会告诉你**具体哪里不对 + 可能是什么原因 + 怎么解决**。照着做就行。
